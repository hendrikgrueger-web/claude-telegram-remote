3
Claude Code CLI Installiere mit: npm install -g @anthropic-ai/claude-code. Benötigt Node.js 18+ und einen Anthropic API Key.
4
Telegram-Account & Bot-Token Erstelle einen Bot via @BotFather in Telegram und notiere den API-Token. Außerdem deine numerische User-ID (z.B. via @userinfobot).

Installation

In 5 Schritten startklar

Das Setup-Script erledigt den Rest automatisch.

1

Repository klonen

git clone https://github.com/hendrikgrueger-web/claude-telegram-remote.git
2

In das Verzeichnis wechseln

cd claude-telegram-remote
3

Setup starten

bash install.sh
Das Script erstellt automatisch ein Python-Virtualenv, installiert alle Abhängigkeiten, sucht die Claude CLI auf deinem System und richtet den macOS launchd-Service ein, damit der Bot im Hintergrund läuft.
4

Bot-Token & User-ID eingeben

Das Setup-Script fragt interaktiv nach deinem Telegram Bot-Token (von @BotFather) und deiner numerischen Telegram User-ID. Diese werden sicher in der lokalen Konfiguration gespeichert.

5

Auto-Login (optional — nur für 24/7-Betrieb ohne Anmeldung)

Auto-Login ist nicht zwingend nötig für den normalen Betrieb. Der Bot läuft automatisch, solange du eingeloggt bist. Nach einem Neustart genügt es, dich ganz normal anzumelden — der launchd-Service startet den Bot dann automatisch im Hintergrund.

Nur wenn der Mac nach einem Neustart ohne jede Interaktion den Bot starten soll, aktiviere Auto-Login: Systemeinstellungen → Allgemein → Automatisch anmelden.

Installation

Ready in 5 steps

The setup script takes care of the rest automatically.

1

Clone the repository

git clone https://github.com/hendrikgrueger-web/claude-telegram-remote.git
2

Enter the directory

cd claude-telegram-remote
3

Run the setup

bash install.sh
The script automatically creates a Python virtualenv, installs all dependencies, locates the Claude CLI on your system, and sets up the macOS launchd service so the bot runs in the background.
4

Enter bot token & user ID

The setup script interactively asks for your Telegram bot token (from @BotFather) and your numeric Telegram user ID. These are stored securely in the local configuration.

5

Auto-Login (optional — only for unattended 24/7 operation)

Auto-Login is not required for normal use. The bot runs automatically as long as you are logged in. After a reboot, simply log in as usual — the launchd service will start the bot automatically in the background.

Only if you want the Mac to start the bot after a reboot without any interaction, enable Auto-Login: System Settings → General → Automatic Login.

Troubleshooting

Probleme & Lösungen

Die häufigsten Probleme und wie du sie behebst.

Bot antwortet nicht

Prüfe die Log-Datei auf Fehlermeldungen. Dort siehst du, ob der Service läuft und was schiefgelaufen ist.

tail -f ~/Library/Logs/claude-telegram/bot.log

Session abgelaufen

Claude Code Sessions haben eine Lebensdauer. Wenn eine Session abläuft, wird sie automatisch neu gestartet — einfach die letzte Nachricht nochmal senden.

Timeout bei langen Anfragen

Manche Aufgaben dauern länger als erwartet. Brich die laufende Anfrage ab und versuche es erneut — ggf. mit einer kleineren Teilaufgabe.

/stop

claude: No such file or directory

Der Bot findet die Claude CLI nicht. Setze den Pfad explizit in deiner .env-Datei. Du kannst den Pfad mit which claude herausfinden.

CLAUDE_BIN=/path/to/claude

afk-mode beta header error

Dieser Fehler tritt auf, wenn experimentelle Beta-Features aktiv sind. Er ist bereits standardmäßig behoben: Die Variable CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 ist in der .env.example voreingestellt und wird beim Setup übernommen.

Troubleshooting

Problems & Solutions

The most common issues and how to fix them.

Bot does not respond

Check the log file for error messages. There you can see whether the service is running and what went wrong.

tail -f ~/Library/Logs/claude-telegram/bot.log

Session expired

Claude Code sessions have a lifespan. When a session expires, it is automatically restarted — just resend the last message.

Timeout on long requests

Some tasks take longer than expected. Cancel the running request and try again — perhaps with a smaller subtask.

/stop

claude: No such file or directory

The bot cannot find the Claude CLI. Set the path explicitly in your .env file. You can find the path with which claude.

CLAUDE_BIN=/path/to/claude

afk-mode beta header error

This error occurs when experimental beta features are active. It is already fixed by default: The variable CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 is preset in .env.example and is applied during setup.
