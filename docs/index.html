Installation

Ready in 5 steps

The setup script takes care of the rest automatically.

1

Clone the repository

git clone https://github.com/hendrikgrueger-web/claude-telegram-remote.git
2

Enter the directory

cd claude-telegram-remote
3

Run the setup

bash install.sh
Automatically creates a Python virtualenv, installs all dependencies, locates the Claude CLI, and sets up the macOS launchd service.
4

Enter bot token & user ID

The script interactively asks for your Telegram bot token (from @BotFather) and your numeric Telegram user ID. These are stored securely in the local .env file.
5

Auto-Login (optional)

The bot runs automatically as long as you are logged in. After a reboot, simply log in — launchd starts the bot in the background. Only for unattended 24/7 operation: System Settings → General → Automatic Login.

Sicherheit

Security by Design

Mehrere Sicherheitsebenen schuetzen deinen Mac vor unbefugtem Zugriff.

Wichtig: Der Bot gibt dir vollen Terminal-Zugriff auf deinen Mac. Deshalb ist er so gebaut, dass nur du ihn nutzen kannst. Jede Nachricht wird gegen deine Telegram User-ID geprueft — kein anderer Nutzer kann mit dem Bot interagieren.

Single-User Authentifizierung

Nur deine Telegram User-ID wird akzeptiert. Jede Nachricht von anderen Nutzern wird ignoriert und geloggt.

Dateiberechtigungen

Konfigurationsdateien (.env, workspaces.json) werden mit chmod 600 geschuetzt — nur dein Benutzer kann sie lesen.

Generische Fehlermeldungen

Bei Fehlern werden keine internen Details an Telegram gesendet. Stack-Traces und Pfade bleiben im lokalen Log.

Input-Validierung

Nachrichtenlaenge wird begrenzt (4.000 Zeichen). Keine Shell-Injection moeglich — der Bot nutzt asyncio.create_subprocess_exec statt Shell-Aufrufe.

Zugriffs-Logging

Jeder unbefugte Zugriffsversuch wird mit User-ID, Username und vollstaendigem Namen geloggt — fuer Audit und Nachverfolgung.

Pinned Dependencies & Atomic Writes

Alle Python-Abhaengigkeiten sind auf exakte Versionen gepinnt. Konfigurationsdateien werden atomar geschrieben (write-to-temp + rename).

Security

Security by Design

Multiple security layers protect your Mac from unauthorized access.

Important: The bot gives you full terminal access to your Mac. That's why it's built so that only you can use it. Every message is checked against your Telegram user ID — no other user can interact with the bot.

Single-User Authentication

Only your Telegram user ID is accepted. Every message from other users is silently ignored and logged.

File Permissions

Configuration files (.env, workspaces.json) are protected with chmod 600 — only your user account can read them.

Generic Error Messages

No internal details are sent to Telegram on errors. Stack traces and file paths stay in the local log file.

Input Validation

Message length is limited (4,000 characters). No shell injection possible — the bot uses asyncio.create_subprocess_exec instead of shell calls.

Access Logging

Every unauthorized access attempt is logged with user ID, username, and full name — for audit and tracking.

Pinned Dependencies & Atomic Writes

All Python dependencies are pinned to exact versions. Config files are written atomically (write-to-temp + rename).

Troubleshooting

Probleme & Loesungen

Die haeufigsten Probleme und wie du sie behebst.

! Bot antwortet nicht

Pruefe die Log-Datei auf Fehlermeldungen:

tail -f ~/Library/Logs/claude-telegram/bot.log

! Session abgelaufen

Claude Code Sessions haben eine Lebensdauer. Wenn eine Session ablaeuft, wird sie automatisch zurueckgesetzt — einfach die letzte Nachricht nochmal senden.

! Timeout bei langen Anfragen

Brich die laufende Anfrage ab und versuche es erneut — ggf. mit einer kleineren Teilaufgabe.

/stop

! claude: No such file or directory

Der Bot findet die Claude CLI nicht. Setze den Pfad explizit in deiner .env-Datei:

CLAUDE_BIN=/path/to/claude    # Finden mit: which claude

! afk-mode beta header error

Bereits standardmaessig behoben: CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 ist in .env.example voreingestellt und wird beim Setup uebernommen.

Troubleshooting

Problems & Solutions

The most common issues and how to fix them.

! Bot does not respond

Check the log file for error messages:

tail -f ~/Library/Logs/claude-telegram/bot.log

! Session expired

Claude Code sessions have a lifespan. When a session expires, it is automatically reset — just resend the last message.

! Timeout on long requests

Cancel the running request and try again — perhaps with a smaller subtask.

/stop

! claude: No such file or directory

The bot cannot find the Claude CLI. Set the path explicitly in your .env file:

CLAUDE_BIN=/path/to/claude    # Find with: which claude

! afk-mode beta header error

Already fixed by default: CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS=1 is preset in .env.example and applied during setup.
